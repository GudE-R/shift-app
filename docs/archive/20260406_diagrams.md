## 6. シーケンス図: AIシフト生成

> 参照元: 要件定義書 §6.4.1

```mermaid
sequenceDiagram
    actor Owner as オーナー
    participant App as ShiftCraft
    participant API as API Server
    participant Gemini as Gemini API

    Owner->>App: 「シフト生成」ボタン押下
    App->>API: POST /api/shifts/generate
    API->>API: マスタデータ・条件を収集
    API->>API: プロンプト構築
    API->>Gemini: 構造化プロンプト送信
    Gemini-->>API: シフト案（JSON）

    alt 生成成功
        API->>API: バリデーション・警告チェック
        API-->>App: シフト案 + 警告一覧
        App-->>Owner: カレンダーにシフト案を表示
    else 生成失敗 / タイムアウト
        API-->>App: エラーメッセージ
        App-->>Owner: リトライ案内
    end

    Owner->>App: 手動でシフト調整
    App->>App: リアルタイムバリデーション
    Owner->>App: 確定・PDF出力
```

## 7. シーケンス図: AIシフト生成（詳細）

> 参照元: 設計書 §4.1

```mermaid
sequenceDiagram
    actor Owner as オーナー
    participant UI as フロントエンド
    participant API as API Route
    participant DB as PostgreSQL
    participant PB as Prompt Builder
    participant Gemini as Gemini API
    participant Val as Validator

    Owner->>UI: 生成ボタン押下
    UI->>UI: ローディング表示

    UI->>API: POST /api/shifts/[id]/generate

    API->>DB: 店舗情報取得
    API->>DB: スタッフ一覧取得（所属・条件含む）
    API->>DB: 出勤可否取得
    API->>DB: NG日取得
    API->>DB: 必要人員設定取得
    API->>DB: 制約設定取得

    API->>PB: 全データを渡してプロンプト構築
    PB-->>API: 構造化プロンプト（JSON Schema指定）

    API->>Gemini: generateContent(prompt)
    Note over Gemini: 構造化出力モード<br/>（response_mime_type: application/json）

    alt 生成成功
        Gemini-->>API: シフト案 JSON
        API->>API: JSON パース
        API->>Val: バリデーション実行
        Val->>Val: NG日チェック
        Val->>Val: 勤務時間チェック
        Val->>Val: 連続勤務チェック
        Val->>Val: 人員充足チェック
        Val-->>API: 警告一覧

        API->>DB: 既存AI生成エントリのみ削除（手動分は保持）
        API->>DB: 新AI生成エントリを一括挿入
        Note over API,DB: overwrite_manual=true の場合は全削除

        API-->>UI: シフト案 + 警告 + サマリー
        UI->>UI: カレンダーに反映
        UI-->>Owner: 結果表示 + 警告表示
    else タイムアウト / パースエラー
        Gemini-->>API: エラー or 不正JSON
        API-->>UI: エラーレスポンス
        UI-->>Owner: リトライ案内
    end
```

## 8. シーケンス図: 認証・認可

> 参照元: 設計書 §3.4

```mermaid
sequenceDiagram
    actor Owner as オーナー
    participant Browser as ブラウザ
    participant MW as Middleware
    participant API as API Route
    participant Auth as Supabase Auth
    participant DB as PostgreSQL + RLS

    Owner->>Browser: メール + パスワード入力
    Browser->>Auth: signInWithPassword()
    Auth-->>Browser: Session (JWT)
    Browser->>Browser: Cookie にセッション保存

    Note over Browser,DB: 以降の全リクエスト

    Browser->>MW: リクエスト + Cookie
    MW->>Auth: getUser() でセッション検証
    alt セッション有効
        Auth-->>MW: User情報
        MW->>API: リクエスト転送
        API->>DB: クエリ実行（RLSが自動適用）
        DB-->>API: オーナー自身のデータのみ返却
        API-->>Browser: レスポンス
    else セッション無効/期限切れ
        MW-->>Browser: 302 → /login
    end
```

## 9. シーケンス図: シフト編集（D&D）

> 参照元: 設計書 §4.3

```mermaid
sequenceDiagram
    actor Owner as オーナー
    participant UI as Calendar Grid
    participant DnD as @dnd-kit
    participant Store as Zustand Store
    participant Val as Validator
    participant API as API Route
    participant DB as PostgreSQL

    Owner->>UI: シフトセルをドラッグ開始
    UI->>DnD: onDragStart
    DnD->>UI: ドラッグ中のプレビュー表示

    Owner->>UI: 別のスロットにドロップ
    UI->>DnD: onDragEnd(event)
    DnD->>Store: moveEntry(entryId, newDate, newTime)

    Store->>Val: validate(updatedEntries)
    Val-->>Store: warnings[]

    Store->>Store: Optimistic Update（即時UI反映）
    Store->>API: PATCH /api/shifts/[id]/entries/[entryId]

    alt 保存成功
        API->>DB: UPDATE shift_entries
        DB-->>API: OK
        API-->>Store: 確認
    else 保存失敗
        API-->>Store: エラー
        Store->>Store: ロールバック
        Store-->>Owner: エラー通知
    end

    Store-->>UI: 再レンダリング（警告含む）
    UI-->>Owner: 更新後のカレンダー + 警告
```
