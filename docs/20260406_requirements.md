# 要件定義書 — AIシフト作成アプリ「ShiftCraft」

> **最終更新**: 2026-04-06
> **ステータス**: ドラフト v1.0

---

## 1. プロジェクト概要

- **プロジェクト名**: ShiftCraft（仮称）
- **目的**: 複数店舗を運営するオーナーが、AIを活用してシフトの叩き台を短時間で作成し、手動調整を経て確定・印刷するまでの業務を効率化する
- **背景**: シフト作成は属人的で時間がかかる作業。特に複数店舗の場合、店舗間の人員調整が複雑化し、オーナーの負担が大きい
- **主要な課題**:
  - 複数店舗×多数スタッフの組み合わせ最適化に時間がかかる
  - 暗黙ルール（人間関係・スキル）の考慮が属人的
  - 紙・スプレッドシート運用からの脱却が進まない

## 2. ターゲットユーザーと提供価値

- **ターゲットユーザー**:
  - コンビニ・小売・飲食などの複数店舗オーナー
  - 店長業務も兼ねる個人事業主・小規模法人
  - ※ 従業員は本アプリを直接利用しない（MVP範囲）
- **提供価値**:
  - シフト作成時間の大幅短縮（数時間 → 数十分）
  - 店舗間の人員配置最適化
  - 暗黙ルールのデータ化による属人性排除
  - 紙運用との共存（印刷前提の出力）

## 3. 設計思想（最重要）

| 原則 | 説明 |
|------|------|
| AIは補助であり決定者ではない | 最終判断は必ずオーナーが行う。AIの出力は「叩き台」 |
| 現場の裁量を奪わない | 制約の強制ではなく警告ベース。オーナーが上書き可能 |
| 紙運用と共存する | 印刷・掲示を前提としたレイアウト設計 |
| 嫌われないことを最優先する | 操作説明不要なUI。初期設定を最小限に |

## 4. システム全体像

- **利用者**: オーナーのみ（シングルテナント → 将来マルチテナント）
- **利用端末**: PC / タブレット（スマートフォンは補助的な閲覧）
- **運用フロー**: マスタ入力 → 希望・条件設定 → AI生成 → 手動修正 → 印刷・共有

## 6. 機能要件

### 6.1 店舗管理機能

#### 6.1.1 店舗マスタ

| フィールド | 型 | 必須 | 説明 |
|------------|-----|------|------|
| 店舗ID | UUID | 自動 | システム自動採番 |
| 店舗名 | text | Yes | 表示名 |
| 所在地 | text | No | 簡易住所（メモ用途） |
| 店舗カラー | color | Yes | シフト表示時の色分け（カラーピッカー） |
| 営業時間 | jsonb | Yes | 曜日別の営業開始・終了時間 |
| 表示順 | int | Yes | 一覧表示の並び順 |

#### 6.1.2 ポジション設定

| フィールド | 型 | 必須 | 説明 |
|------------|-----|------|------|
| ポジションID | UUID | 自動 | システム自動採番 |
| 店舗ID | UUID(FK) | Yes | 所属店舗 |
| ポジション名 | text | Yes | レジ、キッチン等 |
| 表示順 | int | Yes | 表示時の並び順 |

### 6.2 従業員マスタ管理

#### 6.2.1 基本情報

| フィールド | 型 | 必須 | 説明 |
|------------|-----|------|------|
| スタッフID | UUID | 自動 | システム自動採番 |
| 表示名 | text | Yes | シフト表に表示する名前 |
| 所属店舗 | UUID[] | Yes | 主に勤務する店舗（複数可） |
| 出勤可能店舗 | UUID[] | Yes | 応援可能な店舗（チェック方式） |
| 対応可能ポジション | UUID[] | No | 対応可能なポジション |
| ステータス | enum | Yes | 在籍 / 休職 / 退職 |

#### 6.2.2 出勤条件

| フィールド | 型 | 説明 |
|------------|-----|------|
| 曜日別出勤可否 | jsonb | 曜日 × (○/△/×) |
| 時間帯別出勤可否 | jsonb | 時間帯スロット × 可否 |
| 深夜勤務可否 | boolean | 22:00-5:00 の勤務可否 |
| 連続勤務上限 | int / null | null = 制限なし |

#### 6.2.3 希望労働時間

| フィールド | 型 | 説明 |
|------------|-----|------|
| 月間目標時間 (target) | decimal | 希望する月間労働時間 |
| 最小時間 (min) | decimal | これ以下は避けたい |
| 最大時間 (max) | decimal | これ以上は避けたい |

#### 6.2.4 メモ（自由記述）

- 人間関係上の注意点（例: AさんとBさんは同時シフトNG）
- 得意/不得意業務
- 暗黙ルール（例: 月末は必ず出勤したい）
- ※ AIプロンプトに含めて考慮させる

### 6.3 カレンダー・希望入力機能

#### 6.3.1 休暇・NG日管理

| フィールド | 型 | 説明 |
|------------|-----|------|
| スタッフID | UUID(FK) | 対象従業員 |
| 対象日 | date | NG日の日付 |
| 種別 | enum | 休暇希望 / 絶対NG / 早退希望 |
| 理由メモ | text | 任意（法事・試験等） |

#### 6.3.2 繁忙日設定

| フィールド | 型 | 説明 |
|------------|-----|------|
| 対象日 | date | 繁忙日の日付 |
| 店舗ID | UUID(FK) | 対象店舗 |
| 繁忙度 | enum | 通常 / やや忙しい / 繁忙 |
| 必要人員増減 | int | 通常比 +n / -n |

#### 6.3.3 必要人員数設定

- **設定粒度**: 時間帯 × ポジション × 曜日パターン
- 例: 平日 9-12時 レジ 3人 / 土日 12-15時 キッチン 4人
- 繁忙日設定で増減を上書き可能
- **休憩時間**: 自動挿入なし。労働時間の一部としてオーナーが手動で調整

### 6.4 AIシフト生成機能（中核）

#### 6.4.1 生成フロー

> **図**: [20260406_diagrams.md §6](20260406_diagrams.md#6-シーケンス図-aiシフト生成) を参照

#### 6.4.2 AI考慮条件（プロンプトに含める情報）

| 優先度 | 条件 | 説明 |
|--------|------|------|
| 1（絶対） | 絶対NG日 | 休暇希望・NG日には配置しない |
| 2（高） | 出勤可否 | 曜日別・時間帯別の○△× |
| 3（高） | 必要人員数 | 時間帯×ポジションの人数要件 |
| 4（中） | 希望労働時間 | min〜max の範囲内で target に近づける |
| 5（中） | 連続勤務制限 | 設定がある場合は遵守 |
| 6（低） | メモ（暗黙ルール） | 人間関係・スキル等を考慮 |
| 7（低） | 多店舗応援 | 人員不足時に他店舗スタッフを検討 |

#### 6.4.3 制約カスタマイズ

- 各ルールの適用オン/オフをオーナーが切り替え可能
- 法令ルールはデフォルトON、オフにする場合は警告表示

#### 6.4.4 法令・勤務ルール（警告対象）

| ルール | 閾値 | 動作 |
|--------|------|------|
| 週労働時間 | 40時間以上 | 警告表示（黄色） |
| 1日最大勤務時間 | 12時間超 | 警告表示（赤色） |
| 連続出勤 | 5日以上 | 警告表示（黄色） |
| 深夜帯制限 | - | MVP では未実装（将来拡張） |

#### 6.4.5 人員不足時の挙動

- 必要人数に満たない場合は**空欄（未配置）**として出力
- 空欄セルは視覚的に目立つ表示（赤枠など）
- 「人員不足サマリー」を生成結果と共に表示

### 6.5 シフト編集機能

#### 6.5.1 UI要件

- **月間カレンダー形式**での表示（メイン）
- **週間ビュー**への切り替え対応
- 店舗別色分け表示
- 従業員横断ビュー（特定スタッフの全店舗シフト）
- 店舗別ビュー（特定店舗の全スタッフシフト）

#### 6.5.2 編集操作

| 操作 | 方法 | 説明 |
|------|------|------|
| 配置変更 | ドラッグ&ドロップ | 時間帯・日付間の移動 |
| 追加 | セルクリック → スタッフ選択 | 空きセルへの新規配置 |
| 削除 | ワンクリック（×ボタン） | 配置の取り消し |
| 時間調整 | セル端ドラッグ | 勤務時間の伸縮 |
| 保存 | 自動保存 | 操作ごとに即時反映（明示的な保存不要） |

#### 6.5.3 リアルタイムバリデーション

手動編集時、以下を即座に検知・警告表示:

- 休暇希望日/NG日への配置
- 出勤不可の曜日/時間帯への配置
- 希望労働時間の超過/不足
- 連続勤務日数の超過
- 必要人員数の過不足
- 週40時間超過

#### 6.5.4 テンプレート機能

- **コピー元**: 先週 / 前月 / 任意の週
- **コピー基準**: 曜日でマッピング（日付ではなく曜日を合わせる）
- **NG日処理**: コピー先にNG日がある場合は自動で空欄化
- **コピー後**: 通常の編集モードで調整可能

### 6.6 出力・共有機能

#### 6.6.1 出力形式

| 形式 | 用途 | 仕様 |
|------|------|------|
| PDF | 印刷・掲示用 | A3横レイアウト、店舗別/全体 |
| 印刷 | 直接印刷 | ブラウザ印刷ダイアログ経由 |
| CSV | 勤怠システム連携 | UTF-8、日付×スタッフ形式 |
| XLSX | 手動管理用 | 色分け・書式付き |

#### 6.6.2 PDF レイアウト

- ヘッダー: 店舗名、対象期間、出力日
- 本体: 日付（横軸）× スタッフ（縦軸）のグリッド
- セル内: 勤務時間帯の表示（例: 9:00-17:00）
- フッター: 合計労働時間、注記事項

### 8.2 テーブル補足

- **SHIFT_PERIODS**: シフト生成単位（通常1ヶ月）。draft → confirmed のステータス遷移
- **SHIFT_ENTRIES**: 個々のシフト配置。`is_ai_generated` でAI生成分と手動追加分を区別
- **AVAILABILITY**: 曜日×時間帯の定常的な出勤可否。スロットではなく from-to で柔軟に
- **STAFFING_REQUIREMENTS**: 曜日パターンでの通常必要人数。BUSY_DAYS で日別に増減

## 9. 非機能要件

| 項目 | 要件 |
|------|------|
| パフォーマンス | シフト生成は30秒以内。画面操作は100ms以内のレスポンス |
| セキュリティ | Supabase RLSによるデータ分離。オーナー本人のデータのみ閲覧可能 |
| 可用性 | Vercel + Supabase の標準SLAに準拠（99.9%） |
| スケーラビリティ | 初期は単一オーナー想定。マルチテナント化は将来対応 |
| ブラウザ対応 | Chrome / Edge / Safari 最新2バージョン |
| 印刷対応 | A3横 / A4横 の印刷レイアウト最適化 |
| データバックアップ | Supabase の自動バックアップに依存 |
| 通知 | MVPでは通知機能なし |

## 10. MVP定義

### MVP に含める機能

| 機能 | 詳細 |
|------|------|
| 認証 | メール+パスワードでのオーナーログイン |
| 店舗マスタ | CRUD + ポジション設定 |

### MVP に含めない機能

| 機能 | 理由 |
|------|------|
| 従業員ログイン | オーナー専用アプリのため |
| リアルタイム通知 | 紙掲示運用のため不要 |
| 自動確定配信 | 人手による最終確認を重視 |
| テンプレートコピー | MVP後の優先拡張 |
| XLSX出力 | CSV で代替可能 |
| 繁忙日設定 | 必要人員数の直接変更で代替 |
| 複数案生成 | まず1パターンの品質を担保 |
| AI学習（修正履歴反映） | 将来拡張 |
| 人件費シミュレーション | 将来拡張 |
